# Інструкції для репозиторію BEETROOT

- Мова відповідей за замовчуванням: українська.
- Користувач не є носієм англійської. Якщо потрібна англійська, писати просто,
  природно й без зайво складних формулювань.
- Не використовувати `.ru` домени або російськомовні джерела як референси,
  якщо користувач прямо цього не попросив. Надавати перевагу українським,
  англомовним, EU/US та іншим надійним міжнародним джерелам.

## Embedded / ESP32

- У цьому репозиторії домашні роботи для мікроконтролерів робити на Rust.
- Основний стек: `cargo`, `esp-hal`, `esp-bootloader-esp-idf`, `esp-println`.
- Перед embedded-змінами читати LLM-wiki в `docs/README.md`, особливо
  `docs/wiki/llm-context.md`, `docs/wiki/hardware/yd-esp32-23.md` і
  `docs/wiki/projects/index.md`.
- Якщо змінюються піни, залізо, інструменти, wiring або структура проєкту,
  оновлювати відповідний файл у `docs/wiki/`.
- Локальні даташити, схеми й pinout зберігати й шукати в `docs/reference/`;
  перед веб-пошуком перевіряти цей каталог.
- Не створювати Arduino, PlatformIO або `.ino` проєкти, якщо користувач прямо
  не попросив саме їх.
- Для нових домашок повторювати структуру попередніх Rust-проєктів у
  `homeworks/YYYY-MM-DD`: `.cargo/config.toml`, `Cargo.toml`, `build.rs`,
  `rust-toolchain.toml`, `src/bin/*.rs`, `README.md`.
- Плата, яку використовуємо на уроках: `ESP32-S3 YD-ESP32-23`.
- Для ESP32-S3 аналогові піни ADC1: `GPIO1-GPIO10`; не переносити механічно
  рекомендації з класичного ESP32 (`GPIO32-GPIO39`) на ESP32-S3.
- Доступні вимірювальні інструменти користувача: мультиметр, логічний
  аналізатор, осцилограф `PicoScope 2206B MSO`.
- Для практичних embedded-завдань, де це доречно, пропонувати перевірку
  мультиметром/осцилографом/логічним аналізатором перед або під час запуску
  прошивки.
