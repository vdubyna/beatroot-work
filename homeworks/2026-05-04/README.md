# Домашня робота: LDR, ADC і напруга на ESP32-S3

Rust-проєкт для моніторингу фоторезистора через ADC на `ESP32-S3
YD-ESP32-23`. Основна прошивка читає середню точку LDR-подільника на `GPIO4`
та друкує калібровану напругу в мілівольтах. Три зовнішні світлодіоди показують
рівень освітлення як шкала: зелений - низьке, зелений+жовтий - середнє,
зелений+жовтий+червоний - яскраве.

## Важливо про ADC на ESP32-S3

Для ESP32-S3 аналогові піни відрізняються від класичного ESP32:

- `ADC1`: `GPIO1-GPIO10`;
- `ADC2`: `GPIO11-GPIO20`, але для домашок краще починати з `ADC1`;
- у цьому проєкті використано `GPIO4` (`ADC1_CH3`).

Важлива помилка, яку вже перевірили на платі: не треба читати ESP32-S3 ADC як
простий RAW і вручну рахувати `RAW / 4095 * Uref` без базової калібровки. Без
`enable_pin_with_cal` заземлений пін може показувати приблизно `1.6-1.8V`.

Правильний шлях у `esp-hal`:

```rust
adc_config.enable_pin_with_cal::<_, AdcCalCurve<ADC1<'_>>>(pin, attenuation)
```

Після цього `read_blocking()` повертає вже калібровану напругу в `mV`. Контрольна
перевірка: `GPIO4 -> GND` має давати `0 mV`.

## Підключення

| Компонент | Куди підключити |
| --- | --- |
| Один кінець LDR | `3V3` |
| Другий кінець LDR | середня точка подільника |
| Один кінець резистора `10 kOhm` | середня точка подільника |
| Другий кінець резистора `10 kOhm` | `GND` |
| Середня точка подільника | `GPIO4` |
| Зелений LED через `220-330 Ohm` | `GPIO15` -> LED/resistor -> `GND` |
| Жовтий LED через `220-330 Ohm` | `GPIO16` -> LED/resistor -> `GND` |
| Червоний LED через `220-330 Ohm` | `GPIO17` -> LED/resistor -> `GND` |

```text
3V3 ---- LDR ----+---- GPIO4 / ADC1_CH3
                 |
               10 kOhm
                 |
                GND
```

Якщо світла багато, опір LDR падає, тому напруга на `GPIO4` зростає. Якщо темно,
опір LDR великий, і напруга на `GPIO4` падає ближче до `GND`.

Для більш стабільних вимірювань можна додати конденсатор `0.1 uF` між `GPIO4` і
`GND`.

Пороги в коді:

- зелений: базовий низький рівень;
- жовтий додається від `2000 mV`;
- червоний додається від `2600 mV`.

## Що друкує Serial Monitor

Кожні `100 ms` основна прошивка друкує CSV:

```text
time_ms,u_calibrated_mv,level
1200,2630,bright
1300,2450,medium
```

- `time_ms` - час від старту прошивки;
- `u_calibrated_mv` - калібрована напруга на `GPIO4` у мілівольтах.
- `level` - рівень, який зараз показують світлодіоди: `low`, `medium` або
  `bright`.

## Запуск

З цієї теки:

```sh
cargo build
```

Прошити плату і відкрити monitor:

```sh
cargo run --bin ldr-adc-voltage
```

Runner у `.cargo/config.toml` використовує `espflash` для `ESP32-S3` і порт
`/dev/cu.usbserial-A5069RR4`.

## Перевірка заліза

Перед запуском або при дивних значеннях перевірити мультиметром:

- `GPIO4 -> GND` прямо на металевому піні має бути `0V`;
- середня точка LDR-подільника має змінюватися зі світлом;
- `GND` breadboard і `GND` плати мають бути спільні.
- кожен LED має бути підключений через резистор `220-330 Ohm`.
