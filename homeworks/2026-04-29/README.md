# Домашня робота: кнопки та режими миготіння LED на ESP32

Rust-проєкт для ESP32-S3. Програма керує двома зовнішніми світлодіодами та
двома зовнішніми кнопками:

- зовнішня кнопка-модуль на `GPIO6` перемикає LED у швидкий режим;
- зовнішня кнопка-модуль на `GPIO7` перемикає LED у повільний режим;
- світлодіоди підключені до `GPIO4` та `GPIO5`;
- миготіння зроблене через `delay`;
- після спрацювання кнопки є debounce-затримка `120 ms`;
- у Serial Monitor друкується вибраний режим.

Після старту програма одразу працює у повільному режимі. Натисни зовнішню
кнопку на `GPIO6`, щоб перейти у швидкий режим. Натисни зовнішню кнопку на
`GPIO7`, щоб повернутися у повільний режим.

## Деталі

| Деталь | Кількість | Примітка |
| --- | ---: | --- |
| ESP32-S3 dev board | 1 | наша плата `YD-ESP32-23` |
| Breadboard | 1 | для зручного макету |
| LED червоний | 1 | звичайний 3/5 мм |
| LED синій або зелений | 1 | другий зовнішній світлодіод |
| Резистор 220-330 Ом | 2 | по одному послідовно з кожним LED |
| Кнопка-модуль з пінами `VCC / OUT / GND` | 2 | дві зовнішні кнопки: `FAST` і `SLOW` |
| Jumper wires | 8-10 | для GPIO, GND і breadboard |
| USB-C кабель | 1 | живлення, прошивка, Serial Monitor |

Зовнішня кнопка - це готовий модуль, а не гола тактова кнопка. На платі
модуля є підписи `VCC`, `OUT`, `GND`. Для ESP32 живи такі модулі від `3V3`, не
від `5V`, щоб сигнал `OUT` був безпечним для GPIO.

## Підключення LED

| ESP32 | Через що | Куди |
| --- | --- | --- |
| `GPIO4` / пін `4` | резистор 220-330 Ом | довга ніжка першого LED |
| коротка ніжка першого LED | дріт | `GND` |
| `GPIO5` / пін `5` | резистор 220-330 Ом | довга ніжка другого LED |
| коротка ніжка другого LED | дріт | `GND` |

Не підключай LED напряму до GPIO без резистора.

## Підключення кнопок

| Кнопка | Пін модуля | Куди підключити |
| --- | --- | --- |
| Кнопка `FAST` | `VCC` | `3V3` на ESP32 |
| Кнопка `FAST` | `OUT` | `GPIO6` / пін `6` на ESP32 |
| Кнопка `FAST` | `GND` | `GND` на ESP32 |
| Кнопка `SLOW` | `VCC` | `3V3` на ESP32 |
| Кнопка `SLOW` | `OUT` | `GPIO7` / пін `7` на ESP32 |
| Кнопка `SLOW` | `GND` | `GND` на ESP32 |

Для цих модулів окремий резистор не потрібен: вони вже мають свою маленьку
плату з обв'язкою. У коді обидві зовнішні кнопки читаються як active-low:
натиснуто = `LOW`, не натиснуто = `HIGH`. Для стабільності на `GPIO6` і
`GPIO7` увімкнено внутрішній `Pull::Up`.

## Як працює програма

Світлодіоди блимають по черзі: спочатку LED на `GPIO4`, потім LED на `GPIO5`.
Кнопки змінюють тільки швидкість:

| Режим | Хто вмикає | LED ON | LED OFF |
| --- | --- | ---: | ---: |
| `FAST` | кнопка-модуль `FAST`, `OUT` на `GPIO6` | `100 ms` | `100 ms` |
| `SLOW` | кнопка-модуль `SLOW`, `OUT` на `GPIO7` | `600 ms` | `600 ms` |

У коді є невелике опитування кнопок кожні `10 ms`, тому натискання зазвичай
ловиться навіть під час `delay`. Режим перемикається саме в момент натискання
і запам'ятовується після відпускання кнопки. Після валідного натискання
програма робить ще одну затримку `120 ms`, щоб прибрати брязкіт контактів.

Serial Monitor показує поточний режим, наприклад:

```text
Boot: button LED modes homework
LEDs: red GPIO4, blue GPIO5
FAST button module: VCC -> 3V3, OUT -> GPIO6, GND -> GND
SLOW button module: VCC -> 3V3, OUT -> GPIO7, GND -> GND
Mode: SLOW (LED on/off 600 ms)
Mode: FAST (LED on/off 100 ms)
Mode: SLOW (LED on/off 600 ms)
```

## Запуск

З теки цієї домашки:

```sh
cargo build
```

Прошити плату і відкрити Serial Monitor:

```sh
cargo run --bin button-led-modes
```

Конфігурація налаштована на порт `/dev/cu.usbserial-A5069RR4`. Якщо macOS
покаже інший порт, зміни його в `.cargo/config.toml`.
