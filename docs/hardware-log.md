# Журнал проєкту ESP32

Дата старту: 2026-05-01

Цей документ ведемо як технічний журнал: що робимо, яке залізо є в наявності, які припущення вже підтверджені, а що ще треба перевірити на платі або в коді.

## Ціль проєкту

Учбовий проєкт на базі ESP32 з основною мовою програмування Rust.

## Джерела інформації

- Фото плати від користувача: `IMG_4484.HEIC`
- Фото зворотної сторони плати від користувача: `IMG_4485.HEIC`
- Перевірка підключеної плати через macOS serial/USB та `espflash`.

## Інструменти для вимірювань

| Інструмент | ПЗ / драйвер | Статус |
| --- | --- | --- |
| Логічний аналізатор USB | Saleae Logic 2 | Є в наявності, підключення перевірено, використовуємо для подальших вимірювань |

Нотатка: PulseView / `sigrok-cli` не вдалося стабільно використати з цим аналізатором на macOS. Для практичної роботи обрано Saleae Logic 2.

## Поточні припущення

- Ймовірно це плата на базі ESP32-S3, бо модуль має маркування `S3-N16R8`.
- `N16R8` ймовірно означає 16 MB flash і 8 MB PSRAM. Flash 16 MB вже підтверджено `espflash`; PSRAM ще треба перевірити окремо.
- Один USB-C порт може бути нативним USB/OTG ESP32-S3, інший може йти через FTDI як USB-UART. Це треба перевірити підключенням до комп'ютера.
- Для першої прошивки варто використовувати порт `COM`, бо він ймовірно підключений через FTDI USB-UART.
- Порт `USB` / `OTG-USB` ймовірно є нативним USB ESP32-S3. Його можна буде перевірити пізніше, але для старту краще не змішувати з прошивкою через `COM`.
- Вбудований RGB LED у прикладі `examples/one-led` налаштований як WS2812-style LED на `GPIO48`. Це треба підтвердити фізично: LED має блимати червоним раз на секунду.

## Що треба перевірити

- Точну модель плати й посилання на документацію або pinout.
- Який USB-C порт використовується для прошивки через UART, а який для native USB / OTG.
- Який GPIO фактично підключений до RGB LED, якщо `GPIO48` не блимає.
- Чи доступні 8 MB PSRAM у прошивці.
- Порт `USB` / `OTG-USB` як native USB ESP32-S3.

## Наступні кроки

- Підтвердити очима, чи блимає вбудований RGB LED після `examples/one-led`.
- Якщо LED не блимає, знайти правильний GPIO для RGB LED.
- Перевірити wiring кнопки на `GPIO1`: у ненатиснутому стані прошивка має показувати `HIGH idle`.
- Перевірити PSRAM.
